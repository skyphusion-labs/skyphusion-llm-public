            assemble bundle
            
        
      
            
      
            
        upload training images for each character slot that the validated
        storyboard locked in (usually 8 or more per slot for LoRA training).
        each file is staged to R2 immediately. when every slot has staged
        images, click "bundle" to assemble the .tar.gz the GPU worker pulls.
      
            
      
            
      
            
        
        
      
            
      
            
    
            

    
    
            
      
            
        
            render
            
        
        
      
            
      
            
        submit the bundle to the vivijure-serverless RunPod endpoint. the
        worker pulls it, trains LoRAs, renders SDXL keyframes plus Wan I2V
        motion, exports a silent MP4 to R2. the status pane polls every 8
        seconds until the job reaches COMPLETED, FAILED, CANCELLED, or
        TIMED_OUT.
      
            
      
      
      
      
      
            
        
        
        
      
            
      
            
        advanced settings (optional)
        
            
          identity & face
          
            
            
            
            
          
            
          
          
            
            face lock + InstantID (advanced)
            
            
              
              
              
              
              
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as face_lock_overrides (with a nested
              instantid sub-block on the wire). controls the
              face-lock pipeline that runs alongside the per-slot LoRA.
              bumping ip_adapter_scale toward 1.0 is the
              quickest way to push identity strength when a LoRA is weak.
            
            
          
            
          
          
            
            multi-character composite (advanced)
            
            
              
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as multi_character_overrides. controls how
              SDXL keyframes for scenes with 2+ cast slots get rendered. the
              regional engine (default) does a single SDXL pass with per-region
              IP-Adapter masks; lora=0.3 and ip=0.7 are the v15-confirmed
              identity-routing settings. composite_legacy is the pre-Phase-R
              panel + grabcut + tile path, kept as an escape hatch. leave a
              field blank to use the pod's compiled default for that key
              instead.
            
            
          
            
        
            
        
            
          video & motion (Wan)
          
          
            
            render output + Wan (advanced)
            
            
              
              
              
              
              
              
              
              
            
            
            
            
              written into render_overrides on the wire. lower
              wan_inference_steps trades quality for speed;
              wan_guidance_scale controls prompt adherence (low
              = creative, high = literal). keyframe_sdxl_size is
              the SDXL bucket SDXL renders the still in - 1216x832 is the
              v0.4.7 multi-subject coherence fix.
            
            
          
            
          
          
            
            Wan diffusion model + inference (advanced)
            
            
              
              
              
              
              
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as wan_diffusion_overrides. these
              override the base wan_diffusion block in
              config.yaml. the flat wan_inference_steps
              / wan_num_frames render_overrides above still
              win when both are set since they apply after the quality
              tier's per-shot adjustment.
            
            
          
            
        
            
        
            
          image & SDXL
          
            
            
            
          
            
          
          
            
            SDXL base + keyframe + seeds (advanced)
            
            
              
              
              
              
              
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as local_diffusion_overrides +
              generation_overrides. SDXL base controls govern
              the img2img path used for portraits / chained scenes;
              keyframe controls govern the CFG-capable SDXL base used for
              the per-shot keyframe (turbo refused at the keyframe stage).
              seed_mode here is the block-level value;
              seed_mode in render_overrides above wins on
              key conflict.
            
            
          
            
          
          
            
            adetailer hand/face fix (advanced)
            
            
              
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as adetailer_overrides. controls the
              post-SDXL inpaint pass on detected hand / face regions.
              disabling entirely is the fastest win when iterating on
              composition; lower inpaint_strength preserves
              more of the original pixels.
            
            
          
            
        
            
        
            
          LoRA & training
          
          
            
            LoRA training (advanced)
            
            
              
              
              
              
            
            
            
            
              forwarded as lora_train_overrides on the render submit.
              applies to fresh Stage 1 LoRA training during the render only; pre-
              trained cast LoRAs (via /cast) are reused as-is and ignore these.
            
            
          
            
          
          
            
            LoRA quality gate (advanced)
            
            
              
              
              
              
              
              
              
            
            
            
            
              forwarded as quality_gate_overrides. controls the
              post-training similarity probe that scores each LoRA against the
              cast portrait. leave fields blank to use the pod's
              config.yaml defaults.
            
            
          
            
          
          
            
            LoRA training extras + encoding + image profile (advanced)
            
            
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as lora_train_extras +
              loras_overrides + quality_overrides
              + image_models_overrides. training extras cover
              the loras.training keys not on the per-call path
              (enabled / min+max images / trigger_template). encoding knobs
              govern ffmpeg's silent-MP4 assembly. image profile picks
              the SDXL profile when style_category="None".
            
            
          
            
        
            
        
            
          continuity & timing
          
          
            
            consistency + chaining (advanced)
            
            
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as consistency_overrides +
              video_consistency_overrides. these set the pod's
              module-level defaults that affect chain logic, IP-Adapter
              cross-shot identity, and the motion-prompt suffix. leave
              fields blank to use the pod's config.yaml
              defaults.
            
            
          
            
          
          
            
            continuity + prompting (advanced)
            
            
              
              
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as continuity_overrides +
              image_prompting_overrides +
              character_generation_overrides. controls scene
              chaining, the always-appended prompt suffixes that SDXL gets
              for anatomy + identity coherence, and the img2img knobs the
              portrait gen path uses.
            
            
          
            
          
          
            
            scene length + movie (advanced)
            
            
              
              
              
              
              
              
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as scene_length_overrides +
              movie_overrides. scene-length caps gate the
              planner's clip-duration math. movie block governs
              production_mode=movie chain defaults; the flat
              wan_num_frames / wan_inference_steps
              render_overrides keys above still win when both
              are set.
            
            
          
            
        
            
        
            
          pipeline & production
          
          
            
            cast bible + production gates + top-level switches (advanced)
            
            
              
              
              
              
              
              
              
              
              
              
              
              
              
              
            
            
            
            
              forwarded as character_bible_overrides +
              production_overrides +
              top_level_switches. cast bible knobs govern the
              condensed appearance block prepended to every shot;
              production gates control hand/face fixes + reference count
              requirements + LoRA training threshold; top-level switches
              flip overall pipeline behavior.
            
            
          
            
        
            
      
            

      
      
            
        expert: raw JSON
        
            
          prompt templates (JSON)
          
          
            
            optional JSON object of the prompt_templates shape:
            anatomy_positive_base / _human / _anime,
            anatomy_negative_global / _focused / _portrait / _anime,
            portrait_positive, hand_positive,
            hand_negative, framing_hints (array of
            strings cycled per shot), act_mood (act-name to phrase
            map). forwarded as prompt_templates_overrides. malformed
            JSON is ignored. leave empty for the pod defaults.
          
            
        
            
        
            
          raw JSON overrides (wins on conflict)
          
          
            
            optional JSON object merged into the render payload sent to the GPU
            worker. fields here override the bundle's storyboard / config
            defaults for this render only AND win over the structured fields
            above on any key conflict. common keys not exposed above:
            wan_inference_steps (int), wan_num_frames (int),
            style_prefix (string). leave empty for no extra overrides.
